# 基于 STM32 的智能多任务网络天气时钟 

## 项目简介
本项目是一款独立开发、具备高稳定性的智能桌面天气时钟终端。项目基于 STM32F407VET6 纯裸机环境开发，摒弃了传统阻塞式设计，采用**时间片轮询架构**实现多任务并发。通过高度模块化的工程分层设计，成功整合了本地高精度 RTC 走时、AHT20 环境传感、ESP-AT 网络通信与动态 UI 渲染引擎。

## 工程目录与架构设计
本项目严格遵循**硬件抽象层（HAL）与业务解耦**的设计规范：

* **`app/` (应用业务层)**：包含核心调度器 `mloop.c`、页面逻辑 `ui/` 以及字库与图片资源。负责各任务的时间片分发与 UI 渲染调度。
* **`devices/` (外设组件层)**：包含 `espat.c` (Wi-Fi 模块 AT 指令集封装) 与 `weather.c` (HTTP GET 请求与 JSON 数据过滤解析)。
* **`driver/` (硬件驱动层)**：包含 `lcd.c` (基于 SPI2 的 ST7789 驱动与流式传输优化)、`aht20.c` (I2C 温湿度驱动)、`rtc.c` 与 `serial.c`，实现对底层硬件的精准控制。
* **`system/` (系统基准层)**：包含 `delay.c`，利用 SysTick 提供无锁原子级 64 位全局时间戳与非阻塞延时接口。
* **`mdk/` & `firmware/`**：Keil 编译工程文件与 STM32 标准外设库。

## 核心技术亮点与二次重构
虽然部分底层协议与字库剥离自开源组件，但在系统集成中进行了大量的深度优化与重构：

1. **防竞争的时间戳引擎 (`system/delay.c`)**
   针对裸机下频繁中断可能导致的时间戳读取撕裂问题，引入了**无锁双读校验（Sequence Lock）机制**。在不关闭全局中断的前提下，保证了 `cpu_now()` 读取 64 位时钟滴答的绝对原子性。
2. **轻量级排版与差分刷新 (`driver/lcd.c`)**
   重构了字符渲染引擎，彻底抛弃死板的等宽排版。通过读取不同字库（如 32/54/76 号字）的真实物理阵列宽度进行动态步进计算，实现完美的像素级居中与动态边界换行防御；配合 `snprintf` 数据掩码覆盖，以 0 显存开销实现局部差分刷新，消除屏幕闪烁。
3. **多任务非阻塞轮询 (`app/mloop.c`)**
   重写了主循环调度逻辑，将网络请求、传感器采集、UI 刷新等高耗时任务拆分为异步状态机，实现了毫秒级响应的并发调度雏形。

## 后续演进计划 (Roadmap)
- [ ] **V2.0 RTOS 重构**：计划全面引入 FreeRTOS，将现有的时间片轮询状态机升级为抢占式多任务调度，引入互斥量（Mutex）保护 SPI 屏幕总线，进一步提升系统实时性。
